Progress So Far

Over the past few weeks, we have made some initial progress on our project but are admittedly behind schedule relative to our original proposal. We have implemented the sequential greedy graph coloring algorithm and spent time getting used to working with OpenMP and CUDA programming models for graphs. We realized that in our original proposal we only mentioned working with only one parallel graph algorithm and have come to realize why this is not sufficient for the project.

We spent time researching a host of different parallel algorithms we are hoping to implement using OpenMP within the coming weeks. We have also implemented Jones-Plassmann and parallelized it with OpenMP but are still working on experimenting different graphs that parallelization might help with and not help with, and are looking at tradeoffs within this in order to see how we can improve the parallelization of the algorithm.

We also want to work on parallelizing different algorithms for the sequential greedy version, and include things like speculative coloring and conflict resolution and see how this compares to the parallel versions of algorithms we have already implemented.

Preliminary Results

Preliminary results from our OpenMP implementation show a small but measurable improvement in performance. On a graph with 10,000 vertices and approximately 50,000 edges, the sequential algorithm completes in 0.098 seconds, while the OpenMP version (with 8 threads) finishes in 0.087 seconds. The speedup observed is quite modest but leads us to believe we are on the right track with correctness and our initial approach.

Poster Session
